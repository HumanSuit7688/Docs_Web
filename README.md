# Docs_Web: Система автоматического формирования справок об обучении

Веб-приложение на базе Django, разработанное для автоматизации процесса заказа и выдачи справок об обучении в образовательных учреждениях. Проект создан и успешно протестирован на базе **МАОУ Лицей №1 г. Южно-Сахалинска** в рамках региональной инициативы **«Бережная школа»**.

Основная цель проекта — оптимизация внутренней логистики учреждения, сокращение бюрократической нагрузки на секретариат и минимизация времени ожидания справок для учащихся и их родителей.

## 🚀 Ссылка на проект
Приложение развернуто на хостинге Render и доступно по адресу: [https://docs-web-iwxz.onrender.com/](https://docs-web-iwxz.onrender.com/)

---

## ✨ Основные возможности
- **Автоматическая генерация документов:** Динамическое создание официальных справок в формате `.docx` по заранее утвержденному шаблону лицея.
- **Интеллектуальное склонение ФИО:** Интеграция с лингвистической библиотекой для автоматического и корректного перевода ФИО учащегося в дательный падеж (кому выдана справка).
- **Email-оповещения (SMTP):** Автоматическая отправка уведомлений на электронную почту ученика о том, что его справка готова и её можно забрать.
- **Интерфейс секретаря:** Выделенная панель для отслеживания статуса заявок и управления очередью.

---

## 🛠 Стек технологий
- **Backend:** Python 3.x, Django Framework
- **Database:** SQLite3 (хранение и обработка данных моделей `Order`)
- **Document Processing:** `python-docx` (работа с шаблонами Microsoft Word)
- **Linguistics:** `Pytrovich` (автоматическое склонение ФИО)
- **Notifications:** `smtplib` / `email` (отправка писем через SMTP-сервер Yandex)
- **Deployment:** Render Cloud Platform

## 💻 Локальное развертывание и запуск

### 1. Клонирование репозитория
git clone https://github.com/HumanSuit7688/Docs_Web.git
cd Docs_Web

### 2. Настройка виртуального окружения
python -m venv venv
# Для Windows:
venv\Scripts\activate
# Для macOS/Linux:
source venv/bin/activate

### 3. Установка зависимостей
Все необходимые для работы проекта внешние библиотеки и зависимости указаны в файле `requirements.txt`. Для их установки выполните:
pip install -r requirements.txt

### 4. Запуск локального сервера
Запуск проекта осуществляется через стандартный менеджер Django `manage.py`, который находится в корневом каталоге проекта:
python manage.py runserver

После запуска приложение будет доступно локально по адресу: http://127.0.0.1:8000/
